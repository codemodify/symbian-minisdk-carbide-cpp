...What is an OS

- layer that abstracts the hardware.
- a model of how to deal with the resources on a computer.
- software must use this model to work with the resources.
- focuses on providing ways to work with the hardware.

A good system model is one that effectively and transparently
provides software with intuitive ways to access system resources.

Depending on how an OS allows the access to the resources
- using abstraction approach through objects to identify resources,
- or a more direct access from software side
this defines the character of an operating system. That character will evolve
over time as the OS incorporates new technologies.

Parameters that define a good OS:
(o) robustness - reliable and tolerates problems well - provides software services, unless harware fails.
(o) scalability - incorporates resources as they are added to the system.
(o) extensibility - designed to adapt to new technologies that extend the OS beyound the point when it was implemented.
(o) throughput - (work that a CPU can complete in a speciffic time period) - must perform well and achieve high throughput. Must minimize the time spent providing services while maximizing throughput.
(o) portability - able to run the same OS philosophy on different hardware.
(o) security - provide inherent securtity. Must prevent unauthorized users or process from accessing stored data or system services.
